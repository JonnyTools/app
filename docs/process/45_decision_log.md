# Decision Log

Dieses Dokument hält wichtige Projektentscheidungen fest.

## Vorlage

```text
Decision ID:
Datum:
Entscheidung:
Grund:
Auswirkungen:
Status:
```

## Entscheidungen

### D-001 — Facilitator statt starker Admin

Datum: 2026-06-06

Entscheidung:
Die App verwendet ein Facilitator-Modell statt eines starken permanenten Admins.

Grund:
Das System soll fair für alle aktiven Mitglieder sein.

Auswirkungen:
Facilitators können Änderungen vorbereiten, aber nicht allein aktivieren.

Status:
Angenommen

### D-002 — Historie wird nicht überschrieben

Datum: 2026-06-06

Entscheidung:
Historische Daten werden nicht still überschrieben. Korrekturen werden als eigene Einträge gespeichert.

Grund:
Monatsberichte und Fairnessanalysen müssen nachvollziehbar bleiben.

Status:
Angenommen
