# Issue Tracking

## Tool-Empfehlung

Einfacher Start:

```text
GitHub Issues + GitHub Projects
```

## Issue-Struktur

Jedes Issue enthält:

```text
Titel
Beschreibung
Ziel
Akzeptanzkriterien
Priorität
Status
Betroffener App-Kontext
```

## Beispiel

```text
Titel:
Aufgabe als erledigt markieren

Akzeptanzkriterien:
- Mitglied sieht offene Aufgabe.
- Mitglied kann auf „Erledigt“ klicken.
- Ausführungsdatum wird gespeichert.
- Preis-Snapshot wird gespeichert.
- Audit-Log wird erstellt.
```
