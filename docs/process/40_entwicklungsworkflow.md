# Entwicklungsworkflow

## Repository

Empfehlung:

```text
main
develop
feature/*
bugfix/*
release/*
```

## Branch-Regeln

```text
main:
stabile Version

develop:
nächste Version

feature/*:
einzelne neue Funktion

bugfix/*:
Fehlerbehebung
```

## Pull Requests

Jede Änderung sollte über Pull Request laufen.

Ein Pull Request enthält:

- Ziel der Änderung
- betroffene Dateien
- Testhinweise
- Screenshots bei UI
- Link zu Issue
