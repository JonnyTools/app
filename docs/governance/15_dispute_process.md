# Streitfall- und Korrekturprozess

## Zweck

Fehler oder Meinungsverschiedenheiten müssen nachvollziehbar gelöst werden.

## Beispiele für Streitfälle

- Aufgabe wurde als erledigt markiert, aber nicht gemacht
- falscher Preis wurde verwendet
- Ersatzaufgabe wurde nicht abgesprochen
- Zusatzaufgabe ist umstritten
- Monatsbericht enthält Fehler

## Prozess

```text
1. Mitglied erstellt Korrektur- oder Streitfall.
2. Betroffene Daten werden markiert.
3. Aktive Mitglieder oder betroffene Personen prüfen.
4. Entscheidung wird genehmigt oder abgelehnt.
5. Ergebnis wird dokumentiert.
```

## Korrekturen

Korrekturen überschreiben nicht heimlich alte Daten.

Stattdessen entsteht ein Korrektureintrag:

```text
Originalwert
Korrekturwert
Grund
Erstellt von
Genehmigt von
Zeitpunkt
```
