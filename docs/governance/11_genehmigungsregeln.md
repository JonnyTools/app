# Genehmigungsregeln

## Ziel

Nicht jede Aktion braucht dieselbe Zustimmung. Das System soll fair, aber nicht unnötig kompliziert sein.

## Genehmigungslevel

### Level 1 — Keine Genehmigung

Gilt für:

- eigene Aufgabe als erledigt markieren
- eigenes Profil bearbeiten
- eigene Aufgabenhistorie ansehen

### Level 2 — Betroffene Person bestätigt

Gilt für:

- Aufgabe für andere Person erledigen
- Ersatzaufgabe beanspruchen

Beispiel:

```text
Anna war zuständig.
Jonathan erledigt die Aufgabe.
Anna bestätigt, dass Jonathan die Aufgabe übernommen hat.
```

### Level 3 — Mehrheit der aktiven Mitglieder

Gilt für:

- neue Aufgabe hinzufügen
- Aufgabe deaktivieren
- feste Zuständigkeit ändern
- Mitglied deaktivieren

### Level 4 — Einstimmige Zustimmung

Gilt für:

- Preise ändern
- Monatsbericht finalisieren
- alte Berichte korrigieren
- Genehmigungsregeln ändern

## Selbstgenehmigung

Mitglieder dürfen eigene strittige Ansprüche nicht allein genehmigen.

Beispiel:

```text
Jonathan trägt eine Zusatzaufgabe ein.
Jonathan darf diese nicht allein freigeben.
```
